# Format plugins are self-contained and registry-driven

Tapscore formats are source-level plugins: one authoritative server registration owns setup planning, validation, allowance derivation, scoring, result metrics, and serializable client metadata. Generic mobile and static-fixture renderers consume the registered descriptor and structured results; a format may add one separately registered mobile client adapter only when its interactive setup, score-entry, or results UI cannot be expressed generically. Ball-creation strategies deliberately remain a separate registry because they own reusable ball composition and handicap derivation rather than format scoring. This trades automatic runtime plugin loading for strong compile-time integration while preserving the core acceptance test: deleting the format module and its registration removes every production trace of that format.
